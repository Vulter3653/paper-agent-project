# Abstract

Business-school literature review requires more than topically plausible recommendations. Researchers must verify DOI metadata, enforce journal-selection policies, inspect open-access status, and retain reproducible evidence explaining why each paper was included or excluded. A single large-language-model (LLM) response can produce useful suggestions, but it can also return unverifiable metadata and opaque recommendations. We present **Paper Agent**, a traceable multi-agent system for verified scholarly paper discovery. The system decomposes literature review into a 12-stage tool-using pipeline covering planning, journal selection, retrieval, journal filtering, metadata verification, open-access checking, storage, evaluation preparation, relevance scoring, ranking, critic review, and report delivery. The deployed workflow uses a configured scholarly search backend with Web of Science as the primary provider and OpenAlex as a fallback, Crossref for DOI and metadata verification, Unpaywall for open-access status, Cloudflare Workers for execution, and Cloudflare D1 for persistent traces.

We evaluate the system with **Paper-Agent-Bench**. Quantitative comparison is restricted to a verified controlled T001-T003 layer containing nine comparison rows. On this layer, the Single LLM baseline has higher Precision@5 and NDCG@5, while Paper Agent preserves DOI accuracy, strict top-journal precision, traceability, and explicit policy enforcement. We additionally report artifact-only execution evidence: a gated T004-T006 dry-run produced three job rows and 50 result rows, and a staged T007-T012 attempt produced 87 result rows for T008-T012 while T007 timed out after approximately 250 seconds. Later batches were not started. These expansion artifacts are not benchmark validation. Full T004-T020 validation remains incomplete, and D1 batch-aware persistence is not implemented. The results motivate an evaluation approach that treats auditability and honest claim boundaries as first-class system properties.
